## 五、关键业务规则

| 规则 | 来源 | 实现位置 |
|------|------|---------|
| 微信 code 只能使用一次，换取 openid 后失效 | 微信官方 | `auth.service.ts`：调用 `jscode2session` |
| 首次微信登录自动创建 `staff_account`，生成全局唯一 `staff_id` | 需求文档 6.1 节 | `staff.service.ts`：`createIfNotExists` |
| 未绑定手机号不可提交入驻，但可浏览工作台 | 需求文档 9 节 | 阶段 1 只校验登录态，不强制手机号；阶段 2 `intake.submit` 时校验 |
| 隐私授权未勾选不可提交入驻 | 需求文档 9 节 | `account.service.ts`：记录 `privacy_agreed_at` |
| 后台密码哈希存储，不存明文 | 安全要求 | `admin-auth.service.ts`：bcrypt |
| JWT token 过期时间可配置 | 技术方案 | `config.service.ts` 读取 `JWT_EXPIRES_IN` |
| 后台权限点按模块分组，角色可分配多个权限 | 技术方案 9.2 节 | `admin_role_permission` 多对多 |
| `super_admin` 角色不可删除，`code` 硬编码保护 | 通用实践 | `admin-role.service.ts`：delete 时校验 code |
| 字典 `dict_type` 为 `service_category`、`service_area` 时支持树形（`parent_code`） | 技术方案 6.7 节 | `dict.service.ts`：按 parent_code 组装树 |
| 小程序和后台使用两套独立的 JWT 策略，不可混用 | 安全隔离 | 两套 strategy + guard，不同 secret 建议 |

## 六、数据流

### 6.1 小程序登录流程

```text
1. 用户打开小程序，调用 wx.login() 获取 code
2. POST /api/app/auth/wechat-login  { code }
   → 后端调用微信 jscode2session 获取 openid
   → 查询 staff_account：存在则更新 last_login_at，不存在则创建（生成 staff_id）
   → 签发 JWT token（payload: { staff_id, openid }）
   → 返回 { token, staff_id, is_new_user, mobile_bound, privacy_agreed }
3. 小程序存储 token，后续请求自动附带 Authorization: Bearer <token>
4. 如果是新用户，引导手机号绑定 → 隐私授权 → 工作台
5. 如果是老用户，直接进入工作台
```

### 6.2 手机号绑定流程

```text
1. 用户在工作台或资料页点击"绑定手机号"
2. 小程序调用微信手机号授权按钮（open-type="getPhoneNumber"）
3. 获取加密手机号数据后，POST /api/app/auth/bind-phone { encrypted_data, iv }
   → 后端解密手机号（或走短信验证备选方案）
   → 加密存储 mobile_encrypted，计算 mobile_masked
   → 更新 staff_account
4. 返回绑定成功
```

### 6.3 后台登录与权限校验流程

```text
1. 管理员在后台登录页输入用户名+密码
2. POST /api/admin/auth/login → 验证密码 → 签发 admin JWT
   → payload: { admin_user_id, username, roles: ['auditor', 'admin'] }
3. 前端存储 token，后续请求自动附带
4. 每个需要权限的接口：
   → admin-jwt-auth.guard 校验 token 有效性
   → permissions.guard 校验当前用户是否有对应权限点
   → 无权限返回 403
5. GET /api/admin/auth/me → 返回当前用户信息和权限列表（前端用于菜单渲染）
```

## 九、与前后阶段的衔接

```
阶段 0（项目初始化）
  └── 阶段 1（本次：账号、权限、字典）
        ├── 产出：JWT 鉴权体系（小程序端 + 后台端）
        ├── 产出：staff_account 注册逻辑 + staff_id
```
