## 四、数据流：从填写到提交审核

```text
第 1 步：小程序加载资料页
   GET /api/app/profile
   GET /api/app/dicts?types=service_category,service_area,credential_type
   → 如果有已有资料，回填表单；否则展示空表单

第 2 步：用户填写资料，保存（可多次）
   PUT /api/app/profile           { real_name, gender, birth_date, city_code, ... }
   PUT /api/app/profile/skills    { skills: [{category_code, skill_code}] }
   PUT /api/app/profile/service-areas { areas: [{city_code, district_code, area_code}] }
   → 每次保存成功后 toast 提示，但不改变 intake_status

第 3 步：用户上传证件（可多次，与资料填写可交错）
   POST /api/app/files/upload     → file_id
   POST /api/app/credentials      { credential_type, valid_from, valid_to, file_ids }
   → 证件保存后状态为 not_submitted（尚未进入审核）

第 4 步：用户进入提交确认页
   GET /api/app/intake/preview    → 返回资料摘要、证件列表、缺失项清单
   → 前端展示：绿色勾=已填，红色叉=缺失

第 5 步：用户点击提交审核
   POST /api/app/intake/submit
   → 后端校验：手机号已绑定 + 隐私已授权 + 必填资料完整 + 必传证件已上传
   → 通过：staff_intake_status.intake_status = 'under_review'
           各准入证件 audit_status = 'under_review'
           写入 audit_record (action='submit')
   → 不通过：返回缺失项清单，前端展示具体缺少什么
```

## 六、关键业务规则

阶段 2 代码必须遵守以下规则（来自需求文档和 CLAUDE.md）：

| 规则 | 来源 | 实现位置 |
|------|------|---------|
| 未授权手机号不可提交 | 需求文档第 9 节 | `intake.service.ts`：提交前校验 `staff_account.mobile_encrypted` 非空 |
| 未勾选隐私授权不可提交 | 需求文档第 9 节 | `intake.service.ts`：校验 `staff_account.privacy_agreed_at` 非空 |
| 强准入证件（身份证/健康证/无犯罪证明/保险）必须上传才能提交 | 需求文档第 6.3 节 | `intake.constants.ts` 定义必传证件列表，`intake.service.ts` 校验 |
| 证件修改生成新版本进入复审，旧版本保留 | 需求文档第 7 节 | `credential.service.ts`：upsert 时 version+1，旧记录 `current_effective=false` |
| 证件过期默认仅加标签，不自动改 `is_available` | CLAUDE.md 第 70 行 | `staff_listing_status` 表不因 `valid_to < now()` 自动变更 |
| 草稿可反复保存，不进行必填校验；提交时才做全量校验 | 需求文档第 5.1 节 | 保存接口不做必填校验，`submit` 接口做完整校验 |
| 文件上传需校验类型、大小、数量 | 技术方案第 10.3 节 | `file.service.ts`：上传时校验 MIME、`size_bytes`、用户配额 |
| `staff_id` 首次注册时生成，全局唯一，后续不变 | 需求文档第 6.1 节 | `staff_account.staff_id` 创建后不可修改 |
| 敏感字段加密存储（手机号、身份证号、姓名） | CLAUDE.md 数据安全 | `crypto.util.ts` 加解密，写入时加密，读取时解密 |
| 证件图片私有存储，预览走短时 URL | 技术方案第 10.4 节 | `file.service.ts`：生成有时效的签名 URL |

## 七、小程序页面功能说明

### 7.1 资料填写页 (`profile/edit`)

- 基础信息表单：姓名、性别、出生年月、常住城市/区县、工作经验、自我介绍。
- 服务类目多选：调用 `category-picker` 组件，从字典加载。
- 服务区域选择：调用 `area-picker` 组件，三级联动。
- 保存按钮：调用 `PUT /api/app/profile`（分三个接口保存），不触发状态变更。
- 页面卸载时自动保存草稿提示。

### 7.2 服务类目多选组件 (`category-picker`)

- 从字典接口获取服务类目列表（两级：父类目 + 子技能）。
- 支持多选，已选项高亮。
- 选中后展示标签列表，可逐个删除。
- 保存时传 `[{category_code, skill_code}]`。

### 7.3 服务区域三级选择组件 (`area-picker`)

- 三级联动：城市 → 区县 → 商圈（商圈可选）。
- 从字典接口获取区域数据。
- 支持添加多个服务区域。
- 保存时传 `[{city_code, district_code, area_code}]`。

### 7.4 证件上传与管理页 (`credential/index`)

- 按证件类型分组展示：身份证、健康证、无犯罪证明、保险（强准入，标红）、技能证书、学历（可选）。
- 每项显示：证件名称、有效期、审核状态标签、已上传图片缩略图。
- 点击进入 `credential/edit` 页：上传正反面/附件图片、填写证件编号和有效期。
- 上传使用 `wx.uploadFile` → 后端 `/api/app/files/upload`。

### 7.5 提交确认页 (`submit/index`)

- 资料摘要区：展示姓名、手机号（脱敏）、服务类目、服务区域、经验。
- 证件清单区：每项显示是否已上传、是否在有效期内。
- 缺失项清单：红色高亮未填写的必填资料和未上传的强准入证件。
- 全部通过后，提交按钮可点击。
- 提交成功后跳转审核进度页（阶段 3 实现）或工作台。
