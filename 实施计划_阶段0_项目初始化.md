# 阶段 0：项目初始化 — 实施计划与文件结构

> 依据：CLAUDE.md、`家政服务人员入驻_需求文档（独立版）.docx` v1.2、`家政服务人员入驻小程序_开发技术方案.md` v0.1
> 阶段 0 目标：搭建三端工程骨架，产出可运行的空项目、完整 Prisma schema、统一 `.env.example` 和基础工程配置。

> 当前整理说明：`apps/**` 下已有部分 0 字节文件仅视为目录占位，不代表已完成编码。Claude Code 开始编码时，应先补齐对应文件内容；若某个占位路径不再需要，应删除该占位文件并在变更说明中注明。

### 3.2 关键说明

- 阶段 0 生成全部表定义，但阶段 2 只实现标 **粗体** 的表对应的 service/controller。
- 非阶段 2 的表仅存在于 schema 中，确保后续阶段不需要改 schema。
- 所有表默认包含 `id`、`created_at`、`updated_at`，需要软删除的表含 `deleted_at`。
- 索引、外键关系在 schema 中完整定义。

## 四、`.env.example` 占位变量（0.5）

```env
# App
NODE_ENV=development
PORT=3000

# Database
DATABASE_URL=postgresql://USER:PASSWORD@localhost:5432/housekeeping_staff

# JWT
JWT_SECRET=change_me
JWT_EXPIRES_IN=7d

# WeChat Mini Program
WECHAT_APPID=your_appid
WECHAT_APPSECRET=your_appsecret

# File Storage
STORAGE_PROVIDER=local          # local | oss
STORAGE_LOCAL_BASE_PATH=./uploads

# OSS (only when STORAGE_PROVIDER=oss)
OSS_REGION=
OSS_BUCKET=
OSS_ACCESS_KEY_ID=
OSS_ACCESS_KEY_SECRET=
OSS_ENDPOINT=

# Encryption (for sensitive fields)
ENCRYPTION_KEY=change_me_32bytes_base64

# File Preview
FILE_PREVIEW_URL_TTL_SECONDS=300
```
