# 家政服务人员入驻小程序：开发环境与上线准备清单

本文档面向当前项目《家政服务人员入驻独立小程序》阶段一 MVP。目标是先把入驻、资料、证件、审核、后台、消息、日志跑通；预约分销系统、《大家评评理》、支付、保证金、订单系统均后置。

## 1. 你已经具备的条件

- 微信小程序账号和 AppID：已具备。
- Git 仓库：已具备。
- 微信开发者工具：已具备。
- PostgreSQL：已具备。

下一步重点准备：Node.js、包管理器、API 调试工具、数据库客户端、服务器、域名、HTTPS 证书、对象存储。

## 2. 小程序是否需要 HTTPS 域名？

正式环境需要。

微信小程序上线后，`wx.request`、`wx.uploadFile`、`wx.downloadFile`、`wx.connectSocket` 这类网络能力需要配置合法服务器域名。生产接口建议使用类似：

- API 域名：`https://api.yourdomain.com`
- 文件域名：`https://files.yourdomain.com` 或后端统一代理文件访问

开发阶段可以在微信开发者工具里临时关闭域名校验，但这只适合本地调试，不能作为上线方案。

建议：

- 阶段一只先准备一个 API 域名，例如 `https://api.xxx.com`。
- 证件图片上传先走后端接口，再由后端写入 OSS 或本地开发存储。
- 不建议小程序端直接拿 OSS AccessKey 上传文件。

微信后台配置位置：

`微信公众平台 -> 小程序 -> 开发管理 -> 开发设置 -> 服务器域名`

通常需要配置：

- `request 合法域名`：后端 API 域名。
- `uploadFile 合法域名`：如果上传走同一个 API 域名，也填 API 域名。
- `downloadFile 合法域名`：如果下载证件/图片走后端签名链接或代理，也填对应域名。
- `socket 合法域名`：阶段一不需要，除非做 WebSocket。

## 3. 本地开发工具下载

### 3.1 Node.js LTS

用途：后端服务、后台管理端、构建工具、包管理器运行环境。

官方下载：

<https://nodejs.org/en/download>

建议安装 LTS 版本。安装后在 PowerShell 检查：

```powershell
node -v
npm -v
```

### 3.2 pnpm

用途：管理前端、后端依赖。比 npm 更适合 monorepo 项目。

官方安装文档：

<https://pnpm.io/installation>

Windows 推荐方式：

```powershell
npm install --global corepack@latest
corepack enable pnpm
pnpm -v
```

如果后续项目生成了 `package.json`，再在项目目录里固定 pnpm 版本：

```powershell
corepack use pnpm@latest-11
```

### 3.3 VS Code

用途：代码编辑器。如果你主要让 Claude Code 写代码，也建议装，方便查看和手动调整。

官方下载：

<https://code.visualstudio.com/download>

推荐插件：

- ESLint
- Prettier
- Prisma
- Vue 或 React 相关插件，取决于后台前端框架
- WeChat Mini Program 相关语法插件，可选

### 3.4 数据库客户端

你已经有 PostgreSQL，但还需要一个图形化客户端方便看表和数据。

推荐 DBeaver Community：

<https://dbeaver.io/download/>

<https://www.pgadmin.org/download/>

推荐 Apifox，中文环境和接口文档管理更方便：

<https://docs.apifox.com/download/>

官方下载：

<https://www.docker.com/products/docker-desktop/>

## 4. 推荐技术栈

建议让 Claude Code 先做技术方案，不要直接开写。默认可选这套：

- 小程序端：微信原生小程序。
- 后端：Node.js + NestJS 或 Fastify。
- ORM：Prisma。
- 数据库：PostgreSQL。
- 后台管理端：React + Ant Design 或 Vue + Element Plus。
- 文件存储：开发期本地存储，生产期阿里云 OSS 私有 Bucket。
- 部署：ECS + Nginx + PM2 或 systemd。

阶段一不建议引入太多东西：

- Redis：可后置。
- 消息队列：可后置。
- Docker 生产部署：可后置。
- Kubernetes：不要用，过重。

## 5. 阿里云服务器准备

你之前买过阿里云服务器，可以先登录阿里云控制台查看是否还在：

阿里云控制台：

<https://home.console.aliyun.com/>

ECS 产品文档：

<https://www.alibabacloud.com/help/zh/ecs/>

检查路径：

`阿里云控制台 -> 云服务器 ECS -> 实例`

重点确认：

- 实例是否还在。
- 是否已过期。
- 地域在哪。
- 公网 IP 是否存在。
- 系统盘是否够用。
- 安全组是否开放 22、80、443。

### 5.1 推荐服务器规格

开发/测试期：

- 2 核 4G
- Ubuntu 22.04 LTS 或 24.04 LTS
- 系统盘 40G 起
- 带公网 IP

生产初期：

- 2 核 4G 可以起步。
- 文件不要长期放 ECS 本地盘，证件图片建议放 OSS。
- PostgreSQL 可以先放同一台 ECS，但上线后更建议迁到 RDS PostgreSQL。

### 5.2 安全组建议

开放：

- `22`：SSH，仅限你自己的 IP 更安全。
- `80`：HTTP，用于证书申请、跳转 HTTPS。
- `443`：HTTPS，生产接口。

不要开放：

- PostgreSQL `5432` 到公网。
- 后端内部端口，例如 `3000`、`8080`。

后端服务应由 Nginx 反向代理到本机内部端口。

## 6. 域名、备案、HTTPS

### 6.1 是否必须买域名？

上线小程序接口基本需要域名。不要直接用 IP 作为生产 API 地址。

建议准备：

- 主域名：`xxx.com` 或 `xxx.cn`
- API 子域名：`api.xxx.com`
- 文件子域名：`files.xxx.com`，可后置

### 6.2 是否需要备案？

如果使用中国大陆的阿里云 ECS 和大陆域名访问，通常需要 ICP 备案。阿里云备案流程官方说明：

<https://help.aliyun.com/zh/icp-filing/basic-icp-service/user-guide/icp-filing-application-overview>

阿里云文档提到，个人网站备案示例流程预计约 3 到 22 个工作日，实际以管局审核为准：

<https://help.aliyun.com/zh/icp-filing/basic-icp-service/getting-started/quick-start-for-icp-filing-for-personal-websites>

务实建议：

- 如果你面向国内家政服务用户，优先使用大陆服务器 + 备案域名。
- 如果只是内部开发测试，可以先本地调试或用开发工具关闭域名校验。
- 如果用香港/海外服务器，可能省备案，但延迟、合规和小程序审核不确定性更高，不建议作为长期生产方案。

### 6.3 HTTPS 证书

需要给 API 域名配置 HTTPS。

可选方案：

- 阿里云 SSL 证书。
- Let's Encrypt 免费证书。
- 宝塔面板自动证书，适合不熟悉命令行的人，但生产安全配置要谨慎。

阿里云 SSL / 证书产品入口可以从控制台搜索：

`数字证书管理服务` 或 `SSL 证书`

部署方式：

1. 域名解析 `api.xxx.com` 到 ECS 公网 IP。
2. 申请 SSL 证书。
3. 在 Nginx 配置 HTTPS。
4. 访问 `https://api.xxx.com/health`，确认正常。
5. 到微信公众平台配置合法域名。

## 7. 对象存储 OSS

阶段一会上传身份证、健康证、无犯罪证明、保险等敏感材料，不建议长期放在服务器本地目录。

阿里云 OSS 官方文档：

<https://www.alibabacloud.com/help/zh/oss/>

OSS 控制台快速入门：

<https://www.alibabacloud.com/help/zh/doc-detail/31885.htm>

建议配置：

- Bucket 权限：私有。
- 不把 AccessKey 写进小程序端。
- 小程序上传文件到后端，后端再写入 OSS。
- 查看证件图片时，由后端生成短时有效访问链接，或由后端代理返回。
- 证件文件路径按 `staff/{staff_id}/credential/{type}/{version}` 组织。

阶段一如果赶进度：

- 开发环境可先用本地文件目录。
- 生产上线前必须切到 OSS 或等价私有对象存储。

## 8. 生产部署最小架构

推荐最小结构：

```text
微信小程序
  -> https://api.xxx.com
  -> Nginx
  -> 后端 Node 服务
  -> PostgreSQL
  -> OSS 私有 Bucket

后台管理端
  -> https://admin.xxx.com
  -> Nginx
  -> 后台前端静态文件 / 后端 API
```

也可以阶段一先用：

- `https://api.xxx.com`：后端 API
- `https://admin.xxx.com`：后台管理

如果域名暂时只有一个，也可以：

- `https://api.xxx.com/api`
- `https://api.xxx.com/admin`

## 9. 给 Claude Code 开发前准备的材料

请准备好这些信息，但不要把敏感密钥直接发到公共聊天里：

- 小程序 AppID。
- 小程序 AppSecret，后端需要，放 `.env`，不要提交 Git。
- PostgreSQL 连接信息：
  - host
  - port
  - database
  - username
  - password
- Git 仓库地址。
- 是否使用阿里云 OSS。
- 如果用 OSS：
  - endpoint
  - bucket
  - region
  - RAM 子账号 AccessKey
- 服务器信息：
  - ECS 公网 IP
  - SSH 用户
  - 操作系统
  - 域名
- 现有 PRD：
  - `家政服务人员入驻_需求文档（独立版）.docx`

建议 `.env` 示例：

```env
NODE_ENV=development
PORT=3000
DATABASE_URL=postgresql://USER:PASSWORD@localhost:5432/housekeeping_staff
JWT_SECRET=change_me
WECHAT_APPID=your_appid
WECHAT_APPSECRET=your_appsecret
OSS_ENABLED=false
OSS_REGION=
OSS_BUCKET=
OSS_ACCESS_KEY_ID=
OSS_ACCESS_KEY_SECRET=
```

## 10. 建议的实施顺序

### 第一步：本地工具安装

1. 安装 Node.js LTS。
2. 安装 pnpm。
3. 安装 VS Code。
4. 安装 DBeaver 或 pgAdmin。
5. 安装 Apifox 或 Postman。
6. 确认微信开发者工具能打开小程序项目。

### 第二步：让 Claude Code 先出技术方案

建议提示词：

```text
请基于《家政服务人员入驻_需求文档（独立版）》v1.2，先不要编码，先输出开发技术方案：页面清单、后台模块、数据库表、接口清单、状态机、权限模型、文件存储方案、部署方案和分阶段开发计划。当前只做阶段一入驻 MVP，预约分销系统和大家评评理只保留预留接口，不实现完整业务。
```

### 第三步：本地开发 MVP

本地先完成：

- 后端项目初始化。
- 数据库 schema。
- 小程序登录和资料提交。
- 后台审核。
- 证件上传。
- 服务记录摘要人工维护。
- 消息中心。
- 操作日志。

### 第四步：云资源准备

1. 找回或购买 ECS。
2. 准备域名。
3. 做 ICP 备案。
4. 配置 DNS。
5. 配置 HTTPS。
6. 开通 OSS。
7. 配置微信小程序合法域名。

### 第五步：测试与提交审核

上线前检查：

- 小程序隐私政策。
- 用户协议。
- 个人信息收集说明。
- 后台账号权限。
- 证件图片私有访问。
- 数据库备份。
- 操作日志不可随意删除。
- 小程序接口域名全部 HTTPS。

## 11. 当前不建议现在做的事

- 不要现在做支付。
- 不要现在做保证金。
- 不要现在做《大家评评理》。
- 不要现在做完整 C 端预约下单。
- 不要让小程序端直连数据库。
- 不要把 AppSecret、数据库密码、OSS AccessKey 写进前端代码。
- 不要把证件图片放公开读 Bucket。
